# Сайт Георгия Петрова — версия v4

## Что готово

- Форма подключена к Formspree: `https://formspree.io/f/mpqgkvby`.
- Заявки после отправки идут в Formspree и на подтверждённый адрес `1042341@mail.ru`.
- Для формы не нужны Resend, ключи API, серверный код или переменные окружения Vercel.
- Добавлены реальные фотографии: без сгенерированных портретов.
- Переработаны карточки в первом экране: они теперь тёмные, полупрозрачные и читаемые на фото.
- Добавлены фото ключевых кейсов; для ЖК без исходной фотографии оставлена нейтральная архитектурная графика без служебных подписей.
- QR-код и кнопка НРС НОСТРОЙ ведут на официальный поиск по записи Георгия Петрова.

## Проверка формы после публикации

1. Откройте сайт на Vercel.
2. Нажмите **«Обсудить объект»**.
3. Отправьте тестовую заявку с пометкой `ТЕСТ` в тексте.
4. Должна открыться страница `thank-you.html`.
5. Проверьте письмо на `1042341@mail.ru` и раздел Submissions в Formspree.

## Обновление текущего репозитория GitHub

Лучше сначала сохранить текущую версию как резервную копию.

### В корне репозитория заменить файлы

- все файлы `.html`;
- `robots.txt`;
- `sitemap.xml`;
- `README.md`.

### Внутри `assets/js/` заменить

- `config.js`;
- `components.js`;
- `main.js`.

### Внутри `assets/css/` заменить

- `style.css`.

### Внутри `assets/images/` добавить

- все файлы `*.webp` из этой папки;
- `nrs-nostroy-qr.svg` оставить или заменить файлом из этого архива;
- `README.md` можно заменить.

### Внутри `assets/cases/` заменить

- `microdistrict.svg`.

Файлы `api/contact.js` и `.env.example` больше не нужны: Formspree работает напрямую. Их можно удалить, но их присутствие не сломает сайт.

После коммита GitHub Vercel автоматически опубликует обновление.

## Перед покупкой собственного домена

Сейчас canonical, robots и sitemap используют текущий тестовый адрес:

`https://gp-construction.vercel.app`

После подключения собственного домена замените этот адрес в `robots.txt`, `sitemap.xml` и canonical-тегах страниц.

## Что обязательно уточнить до полноценной коммерческой публикации

Заполнить на `privacy.html` сведения об операторе персональных данных: ФИО/ИП/ООО, адрес, контактный e-mail и дату актуализации документа.
