# Сайт Георгия Петрова

Статический адаптивный сайт без сборки: его можно открыть локально, залить в обычный GitHub‑репозиторий и включить GitHub Pages. Ссылки сделаны относительными, поэтому сайт работает и на `username.github.io/repository-name/`, и на собственном домене.

## Что уже есть

- Главная, компетенции, кейсы, четыре страницы ключевых проектов, «О эксперте», контакты, 404 и thank‑you page.
- Структура и тексты рассчитаны на B2B-аудиторию: заказчиков, девелоперов, генподрядчиков и инвесторов.
- Логотип в SVG: не «каска/кран/молоток», а структурный знак из опорного контура, двух управляемых направлений и точки контроля.
- Адаптация под desktop, tablet и mobile; меню, доступная модальная форма, FAQ, UTM-параметры, готовность к Метрике и GA4.
- Встроенная спокойная инженерная графика вместо случайных стоков или нейросетевых «фото объектов».

## Важно до публикации

1. Откройте `assets/js/config.js` и заполните:
   - телефон и e-mail, если они нужны на сайте;
   - номер и официальный URL проверки НРС НОСТРОЙ — только после перепроверки;
   - юридические данные для страницы политики;
   - `formEndpoint` — когда подключите форму;
   - ID Яндекс Метрики / Google Analytics, если они нужны.
2. Замените `https://YOUR-DOMAIN.RU` на реальный домен во всех HTML-файлах, `robots.txt` и `sitemap.xml`.
3. Перед публикацией перепроверьте цифры в кейсах, роль Георгия и право на раскрытие информации по каждому объекту.
4. Обязательно замените визуальные заглушки на реальные фотографии и схемы, где это разрешено. Инструкция — в `assets/images/README.md`.
5. Согласуйте финальную юридическую политику с юристом под фактическую форму работы и подключённые сервисы.

## Как запустить на GitHub Pages

1. На GitHub создайте новый пустой репозиторий, например `petrov-expert-site`.
2. Распакуйте архив и загрузите **содержимое папки** в корень репозитория: `index.html`, `assets`, `robots.txt` и остальные файлы должны лежать в root, а не внутри ещё одной папки.
3. В репозитории откройте **Settings → Pages**.
4. В разделе **Build and deployment** выберите `Deploy from a branch`.
5. Выберите ветку `main` и папку `/ (root)`, затем нажмите **Save**.
6. Через 1–3 минуты GitHub покажет публичный адрес сайта.
7. После подключения домена замените плейсхолдер `YOUR-DOMAIN.RU` и снова загрузите изменённые файлы.

## Как работает форма

Без сервиса формы после заполнения текст обращения копируется в буфер, а Telegram открывается в новой вкладке. Это безопасный временный режим, но не идеальный для заявок.

Чтобы заявки приходили автоматически, в `assets/js/config.js` укажите `formEndpoint`:

```js
formEndpoint: 'https://formspree.io/f/ваш_идентификатор'
```

Подойдут Formspree, Tally с webhook, n8n webhook, Make / Albato или собственный серверный endpoint. После подключения протестируйте форму на телефоне и компьютере.

## Логотип

Файл: `assets/images/brand-mark.svg`.

Знак построен на идее «структуры сложной системы»:
- основной светлый контур — опорная рамка проекта;
- второе направление — связь управленческих контуров;
- небольшой кирпичный квадрат — точка решения / контроля.

Это не буквальная строительная эмблема и не «премиальный» декор. Он работает как компактный знак рядом с типографическим названием и сохраняет читаемость в favicon.

## Редактирование текста и страниц

Каждая страница — отдельный HTML‑файл. Главные точки:

- `index.html` — главная;
- `services.html` — компетенции и форматы работы;
- `cases.html` и `case-*.html` — кейсы;
- `about.html` — профиль;
- `contacts.html` — каналы связи;
- `assets/js/config.js` — данные, контакты, аналитика;
- `assets/css/style.css` — весь дизайн.

## Проверка перед запуском

- Все обязательные поля формы валидируются.
- На мобильном меню открывается и закрывается корректно.
- Ссылки Telegram и MAX перенесены с текущего сайта.
- Не добавляйте формальные обещания строительного контроля, технического заказчика, подписания актов или гарантий ввода, пока договорная модель и полномочия по конкретному объекту не проверены.
